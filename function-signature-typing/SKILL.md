# Function Signature Typing

## 目的

静的型付け言語で関数・メソッドを定義する際、引数と戻り値の型を必ず明示させ、関数シグネチャを契約として扱えるようにする。

## 参照

- ルール本体は `references/rules.md` を参照する。

## 実行手順

1. 作業開始時に `references/rules.md` を読み、対象言語の指針と適用範囲を把握する。
2. 関数・メソッドを実装／修正する際、引数の型と戻り値の型を必ず明示する。ローカル変数の推論は許容する。
3. `as` / `any` / トップ型などで型を弱めず、必要に応じてジェネリクスや `zod` 等の代替手段を使う。
4. 真に推論に任せざるを得ない場合は `references/rules.md` の「ルールから外れる必要がある場合」に従い、ユーザーに許可を求める。
