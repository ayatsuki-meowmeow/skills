# commit-workflow スキルを参照せずコミットした件

- 日付: 2026-05-31
- 関連コミット: `414d0e0` / `61cd91d` / `2241eae`（やり直し後）、`b366821`（破棄した非準拠コミット）

## 発生事象

neverthrow スキル（setup / coding-rules）の変更をコミットする際、リポジトリ内の
`commit-workflow` スキルを参照せずに `git commit` を直接実行した。その結果、1 回目の
コミット `b366821` がスキル規約に対して以下の 3 点（+1）で違反した。

- `Co-Authored-By:` フッターを付けた（スキルは「フッターを付けない」と明示）
- コミット本文を英語で書いた（1 行目サマリは英語でよいが、本文は日本語にすべき）
- `git add -A` で無差別ステージングした（スキルは `git add -A` / `git add .` を禁止）
- 複数の意味単位（setup スキル変更 / coding-rules スキル変更 / eval 挙動証跡）を
  1 コミットに混在させた

## 発生理由

- `commit-workflow` スキルの存在は認識していたのに、コミット着手前に「参照すべきか」を
  確認しなかった。ユーザーの「コミットして」という指示から直接 git コマンド組み立てに進み、
  スキルのトリガー判定を飛ばした。
- スキルの description には「自力で git commit を組み立てる前に必ずこのスキルを参照する」と
  書かれているが、その trigger を能動的に拾えていなかった。

## 再発防止策

- **コミット着手前に必ず `commit-workflow` スキルを参照する。** git コマンドを組む前に規約
  （意味単位分割 / 英語サマリ＋日本語本文の 2 層 / フッターなし / 明示 add）を確認してから実行する。
- 意味単位の分け方が曖昧なときは、分割案を提示してユーザーに仰ぐ。勝手にまとめたり無理に
  分けたりしない。
- やり直しが必要になった場合、未 push であれば `git reset --soft HEAD~1` で安全に巻き直し、
  意味単位ごとに明示ステージングして組み直す。
