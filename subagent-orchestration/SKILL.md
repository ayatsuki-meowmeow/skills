# サブエージェントオーケストレーション

## 目的

メインエージェントのコンテキストを対話と意思決定に集中させ、作業はすべて専門サブエージェントに委譲する。仕様の決定者は常にユーザー。

## 参照

- ルール本体は `references/rules.md` を参照する。開発タスクに着手する前に必ず読むこと。

## 実行手順

1. `references/rules.md` を読む
2. design.md / impl.md を確認する（ドキュメント運用は design-impl-docs スキルに従う）
3. タスクを分解し、rules.md の委譲プロンプト雛形（役割・コンテキスト・タスク・エスカレーション指示・境界）でサブエージェントに委譲する
4. 仕様・要件の疑問が返ってきたら、選択肢と推奨を添えてユーザーに確認し、決定を design.md に記録して作業を再開する
5. 成果を検収し、要約してユーザーに報告する（エスカレーションは報告の先頭に出す）
