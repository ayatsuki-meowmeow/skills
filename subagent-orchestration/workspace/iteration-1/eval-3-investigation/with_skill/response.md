# プラン: Next.js プロジェクトの認証周り調査

サブエージェントオーケストレーション規約に従い、私（オーケストレーター）は直接コードを読まず、調査エージェントに委譲します。今回は「認証周りの現状把握 + リファクタしやすいポイント抽出」という思考・判断が主のタスクなので、**Opus** モデルの調査エージェント 1 体に委譲します（並列化する独立モジュールが事前には見えないため、まずは 1 体で走らせて全体像を掴んでから必要に応じて 2 段目を分割します）。

着手前に、対象タスクの `docs/auth/design.md` と `docs/auth/impl.md` の存在を確認し、無ければ調査エージェントに空ファイルの雛形作成も依頼します（配置規約は design-impl-docs スキルに従う）。調査エージェントには「コードは変更しない」「仕様・要件に関わる疑問（例: 使っている認証プロバイダを今後も踏襲するか、セッション戦略を変えて良いか、など）は自分で決めず未解決の疑問として返す」「成果物末尾に自身のコンテキスト使用率を自己申告する」「50% に近づいたら成果物と未完了リストを明文化して停止する」を必ず含めます。

戻ってきた成果物に仕様・要件の未解決疑問が含まれていた場合は、選択肢とトレードオフを `docs/auth/design.md` の「未決事項」セクションに追記し、ユーザーには「design.md の該当セクションを見て決定してください」とだけ伝えます（チャットに選択肢を長々と貼らない）。純粋な実装リファクタ候補（仕様に影響しない内部設計）は成果物にそのまま列挙してもらい、私は要約してユーザーに報告します。エスカレーションが必要な疑問がある場合は報告の先頭に置きます。

## 委譲プロンプト本文（調査エージェント / Opus）

```
あなたはコードベース調査の専門エージェントです。

【コンテキスト】
- 対象: このリポジトリの Next.js プロジェクト（認証周り）
- ドキュメント: docs/auth/design.md と docs/auth/impl.md
  - まずこの 2 ファイルを読んでください。存在しなければ、design-impl-docs スキルの
    配置規約に従って空の雛形（見出しだけ）を作成してから調査に入ってください。
    - design.md には「概要 / 決定事項 / 未決事項」セクション
    - impl.md には「構成 / 実装状況 / 実装メモ」セクション

【タスク】
現状の認証実装を読み解き、以下を成果物としてまとめてください。
1. 認証周りの現状マップ
   - 認証方式（NextAuth / Clerk / 自前 JWT / Cookie セッション等）と採用箇所
   - 関連ファイルの一覧（app/ or pages/, middleware.ts, lib/auth, API Route 等）
   - リクエスト経路（ログイン、セッション検証、保護されたルート、ログアウト）
   - 認可ロジック（ロール / 権限判定）がどこに散らばっているか
   - サーバー / クライアント境界（Server Component / Client Component / Route Handler / Middleware）
   - 環境変数・外部依存（プロバイダ、DB、キャッシュ等）
2. リファクタしやすいポイントの候補
   - 重複しているセッション取得ロジック
   - 認可判定が複数レイヤーに漏れている箇所
   - middleware と route handler での二重チェック
   - 型が甘い / any が混ざっている箇所
   - テストが手薄で安全にリファクタできる箇所
   各候補について「現状 / 問題 / リファクタ案 / 影響範囲 / リスク」を短く記述。
3. 未解決の疑問
   仕様・要件に関わる疑問（プロバイダ変更の可否、セッション戦略の変更、
   認可モデルの再設計など）は自分で判断せず、選択肢とトレードオフを添えて
   ここに列挙してください。

【境界 / 禁止事項】
- コードの変更は一切しない（調査のみ）
- リファクタの実行もしない（候補提示まで）
- 仕様・要件に関わる判断を独断で行わない → 未解決の疑問に回す
- タスク範囲外のファイル（認証と無関係な機能）に踏み込まない

【コンテキスト使用率の自己申告】
- 成果物の末尾に、自身のコンテキスト使用率（概算 %）を必ず記載してください
- 50% に近づいたら、そこまでの成果と未完了リストを明文化して作業を停止してください
- compact（要約による圧縮）は行わないでください
```
