# 実装 → レビュー反復ループ

## 目的

design.md / impl.md 作成後の実装フェーズを、「実装 → レビュー → 指摘分類 → 記録 → 対応」の反復として回し、レビュー指摘 0 件かつユーザーが動作確認できる状態まで到達させる。

## 参照

- ルール本体は `references/rules.md` を参照する。実装フェーズに入る前に必ず読むこと。
- 役割分担・委譲プロンプト共通要素・エスカレーションプロトコル・モデル選択・コンテキスト管理は subagent-orchestration スキルに従う。
- design.md / impl.md の編集権限・記載範囲は design-impl-docs スキルに従う。
- レビュー実行（本ループの step 3）は code-review-agent スキルに従う（5 レンズ並列 + confidence scoring）。

## 実行手順

1. `references/rules.md` を読む
2. 該当タスクの design.md / impl.md を確認し、実装対象範囲を確定する。反復カウンタ N = 1 で開始
3. 実装エージェントに委譲 → lint チェック（hook 優先、なければオーケストレーター実行、fail 時は修正委譲） → レビューエージェントに委譲 → 指摘を「仕様・設計」と「実装判断」に分類する
4. 仕様・設計は design.md 経由でユーザーへエスカレーション、それ以外は impl.md の「レビュー指摘と対応」に周ごとに記録して対応を決める
5. 対応を実装エージェントに委譲して N++ で 3 に戻る。「指摘 0 件 + lint 通過 + 未決事項なし」を確認したらユーザーに動作確認を依頼して終了。N == 3 で終了条件未達なら design.md 未決事項に状況・選択肢・推奨を記録してユーザーへエスカレーション
