# 認証セッション管理

## 目的・背景

社内向け管理画面の認証基盤。ログイン後のセッション管理を提供する。

## 要件

- ログイン成功時にセッションを発行する
- 有効期限切れのセッションは無効として扱う
- アイドル状態（操作なし）が 15 分続いたセッションは、有効期限内でも無効化する

## 仕様

- セッションはサーバーサイドで管理し、クライアントにはセッション ID のみ渡す
- セッション有効期限は発行から 30 分

## 決定事項

- 2026-06-20: セッションはサーバーサイド管理とする（JWT は採用しない。失効制御を優先）
- 2026-07-03: セッション有効期限を 60 分から 30 分に短縮する（セキュリティ監査の指摘対応）
- 2026-07-03: アイドルタイムアウト 15 分を導入する（同上）
